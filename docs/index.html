Truth-Gated Harness Spec

A starter spec for an AI-agent harness that is best-in-the-world for a narrow purpose, provably — truth-gated, cross-vendor reproduction-and-verification of quantitative work, with a benchmark designed so it can show you not winning.

What this is. A methodology blueprint you can adopt or adapt — not an implementation. It captures the pattern for building a harness where the failure mode is a silently-wrong number, not a crash (the motivating domain is evidence-synthesis methods research: meta-analysis, NMA, DTA, dose-response, registry-borrowing). There is deliberately no proprietary code, no private data, and no secrets here — only the pattern.

The core idea in one screen

One control plane — the Conductor. A light router assigns each unit of work to the strongest live frontier vendor on the right node, gates the result, and moves on. It does not do the heavy reasoning. The router need not be the biggest model (a tested cost lever, not an assumption).
Cross-vendor consensus-or-flag. Every ship verdict is produced by ≥2 independent model families; disagreement flags instead of silently resolving to the optimistic side. Homogeneous agreement is not independent corroboration.
An objective-gate floor under every accept. No ship rests on judges agreeing alone — there is always a named mechanical witness (test / diff / build / reference-parity / runtime check), or the task is reclassified human-in-the-chair.
Reproduction-or-flag. A number is accepted only when an independent vendor re-derives it to declared precision. Verification is re-derivation, not "a reviewer read it."
A private-eval moat — as a pattern. Score yourself against a private ground-truth corpus a competitor can't reproduce (e.g. registry-vs-published gaps), blinded at run time, growing as failures become fixtures. Keep your actual ground truth private; publish only the pattern.
Cost-per-accepted-change, made visible and fenced. A correct harness that costs 10× a single agent for the same output is a science project, not a working program. Efficiency is half the win condition.
Frontier-heterogeneity, qualified on the HARDEST tasks. Weak models collapse on complex, long-horizon work — and the cliff is invisible on easy tasks. Qualify every seat on your hardest tasks; drop any panel member that rubber-stamps.

The 6-stage pipeline

Each stage is a loop with an objective gate, a bounded stop condition, external state, and a fence against its reward-hack.

  1. Problem / idea selection — admissible only if it has a private ground-truth path and a named primary estimand with a reproduction target.
  2. Ground-truth / private-eval acquisition — the moat: an eval nobody else can score against, blinded at run time. Do not publish it.
  3. Method development — frontier models only; spec-driven "make the tests pass" inner loop with an objective gate.
  4. Cross-vendor verification (the crown) — consensus-or-flag on heterogeneous frontier models, an objective-gate floor beneath every pass, reproduce-or-flag, full provenance.
  5. Benchmark — beat published comparators, and run the honest A/B/C harness benchmark on a held-out split.
  6. Publish — ship with the same truth-gates; every claim traces to a source; no placeholder leaks.

The honest benchmark (it can show you NOT winning)

Three arms under an identical budget ceiling, scored on correctness and cost-per-accepted-change, decided on a held-out split the agent never reads:

ArmWhat it isolates
A — Single-agentThe "capable model, brittle scaffold" control.
B — Homogeneous multi-agentDoes more agents alone win, without vendor diversity? If B ≈ C, heterogeneity buys nothing.
C — Heterogeneous truth-gated (yours)Cross-vendor consensus-or-flag + objective-gate floor + reproduction-or-flag, all as Conductor flows.
Designed to be losable. The benchmark must report the honest failure outcomes if they occur: if B ≈ C, heterogeneity isn't paying; if C's cost-per-accepted ≫ A's, it's correct but unaffordable; if C's false-alarm rate is high, it cries wolf. Superiority that only appears on the tuning split is not superiority.

How to adapt it. Swap the domain (any field where the failure mode is a silently-wrong result). Swap the vendors ("family A / B / C" are placeholders for any ≥2 independent frontier families — the differentiator is independence, not any product). And build your own moat: the one part you should not copy is a dataset — there isn't one here.
